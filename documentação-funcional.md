# Tipos de documentações
Aqui será descrito um pouco sobre cada tipo de documentação, trasendo um breve resumo sobre cada uma.

## Tipos de documentação

### Documentação funcional

#### O que é?

documentação que **contextualiza e apresenta** um produto,
funcionalidade ou serviço - a visão geral de um site.

#### Tópicos de estrutura para documentos:

- O que é o produto;
- Breve contexto de seu uso;
- Recursos dísponiveis;
- Pra que (e para quem serve?);
- Principais conceitos (glossário).

### Tutoriais e guias how-to

#### Rápido contexto

Em alguns frameworks de documentação é possível
encontrar **tutoriais e guias how-to's** como sendo
diferentes tipos de documentos

### DIÁTAXIS

- criado por Daniele Procida
- Framework com abordagem de documentação técnica a partir da **qualidade** e **necessidade de usuário**
- Estabelece 4 tipos de documentação e sua relação entre si.

### Semelhanças

- Documentação passo-a-passo
- Guiam o usuário para o que ele precisa **fazer para atingir um objetivo**

  **Tópicos para os documentos**

  - Qual ação preciso realizar?
  - Os passos para chegar até a ação que quero.
 
  citação do Daniele:
 
  > "O **tutorial** serve a necessidade do usuário no
  > estudo. É sua obrigação oferecer uma
  > **experiência de aprendizagem bem sucedida.**
  > O **Guia how-to** serve a necessidade do usuário
  > no trabalho. A sua obrigação é ajudar o
  > usuário a **realizar uma tarefa**"

### Diferenças

| Tutoriais | How-to |
| ------ | ------ |
| Descreve o passo a passo com mais detalhes. | Descreve o passo a passo focando apenas na tarefa a ser realizada.|
|Buscar a dar os porquês de cada ação. | O título come sempre com: "Como fazer alguma coisa"(Ex. como criar uma conta no banco).|
|**Exemplo:** quando se está aprendendo algo pela primeira vez (tipo programação). | Quando você já sabe e quer recapitular como faz.|
